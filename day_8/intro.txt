🐍 Python Class: Understanding try, except, and Error Handling

📌 What is try and except?
In Python, the try and except blocks are used to handle errors and prevent programs from crashing unexpectedly.
When Python encounters an error during execution (called an exception), 
it normally stops the program and shows an error message. 
But using try and except, you can catch those errors, handle them gracefully, and keep the program running.

🎯 Why Do We Use try and except?
✅ To prevent crashes when unexpected input or conditions occur
🔄 To continue program execution even after an error
🧠 To give meaningful messages to the user instead of showing raw errors
🔒 To safely handle sensitive operations like file access, database connection, network requests, etc.

🛠️ Basic Syntax
try:
    # Code that might raise an exception
except ErrorType:
    # Code to run if that error occurs

⚙️ Rules of Using try and except
1. The try block must have at least one except block.
2. You can have multiple except blocks to handle different errors.
3. You can use a generic except: to catch all errors (not recommended for critical applications).
4. You can use else: block – it runs if there was no error in the try block.
5. You can use finally: block – it always runs, even if an error occurs (used for cleanup).

💥 Common Operations to Use with try-except
| Operation             | Possible Error          | Error Type                |
| --------------------- | ----------------------- | ------------------------- |
| Division              | Division by zero        | `ZeroDivisionError`       |
| Input conversion      | Invalid input format    | `ValueError`              |
| List indexing         | Index out of range      | `IndexError`              |
| Dictionary key access | Key not found           | `KeyError`                |
| File reading          | File does not exist     | `FileNotFoundError`       |
| Math functions        | Invalid math operations | `MathError`, `ValueError` |


🧩 Complete Syntax Structure:
try:
    # risky code
except ErrorType1:
    # handle error type 1
except ErrorType2:
    # handle error type 2
else:
    # runs only if no exception
finally:
    # always runs
